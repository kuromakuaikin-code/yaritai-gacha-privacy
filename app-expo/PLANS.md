### 会話フローの書き方（スタイルガイド）

既存データと同じ構造。`q`=自分からの聞き方（丁寧語・実際に口に出せる文）、`a`=相手の答えの例（「◯◯です」形式）、
`d`=聞き返しパターン4つ（タグは既存で使っている「深掘り」「なぜ」「いつ」「誰と」「どのくらい」「逆に」「未来」「過去」「どう」「つなぐ」から選ぶ。最後の1つは必ず「つなぐ」=次のデートや距離を縮める一言）。

- 男女どちらが使っても不自然でない文にする
- 尋問っぽくしない。軽さ・ユーモアを1つは入れる
- d/eカテゴリのセンシティブ話題（恋愛歴・お金）は「自分から開示してから聞く」形を使う（既存のd2参照）

**完成例（a9・この品質で全部書く）:**

```ts
{ id: "a9", title: "音楽の好み・カラオケ", flow: {
  q: "普段どんな音楽聴くんですか？",
  a: "◯◯をよく聴きます",
  d: [["いつ", "いつ聴くことが多いですか？通勤中とか？"],
      ["深掘り", "ライブとかフェスって行くんですか？"],
      ["逆に", "カラオケは歌う派です？聴く専です？"],
      ["つなぐ", "今度おすすめの曲、プレイリストで教えてください"]],
  good: "好きなアーティストの話が出たら「一番好きな曲どれですか？あとで聴いてみます」→次に会う時の話題が1つ確保できる",
  flat: "「私は最近◯◯ばっかり聴いてて…」と自分の話に切り替えて、無理に広げない",
  ng: "「え、それ知らないです」で終わらせる（相手の好きなものを一度受け止めてから質問に変える）",
} },
```

## B. 反応別パターン表（新しい課金の恩恵）

### 型の拡張（`src/topics.ts`）

```ts
export interface TopicFlow {
  q: string; a: string; d: [string, string][];
  good?: string;  // 🌱 盛り上がった時の広げ方
  flat?: string;  // 🍃 反応がうすい時の切り替え方
  ng?: string;    // ⚠ やりがちなNG（カッコ内に理由か言い換えを書く）
}
```

- **既存30話題を含む全52話題**に good / flat / ng を書く（既存話題はIDとq/a/dを変えず、フィールド追加のみ）
- 文はスマホで2行程度まで。上の a9 の例と同じ温度感で

### UIの変更（`src/screens/Topics.tsx` の `FlowBox`）

- `d` パターンの下に「反応別パターン」ブロックを追加:
  - 🌱 盛り上がったら → `good`
  - 🍃 反応がうすい時 → `flat`
  - ⚠ これは避けたい → `ng`
- **プレミアム未購入の場合**: 無料カテゴリの話題でもこのブロックは中身を見せず、
  「⭐ 反応別パターンはプレミアムで見られます」という1行のボタン表示にして、タップで `PaywallSheet` を開く
  （無料ユーザーへの新しい購入導線。q/a/d は今まで通り無料で見せる）
- `FlowBox` は premium 状態を知らないので、props に `premium: boolean` と `onLockedTap?: () => void` を追加して呼び出し側から渡す

### ペイウォール文言の更新（`src/screens/Settings.tsx` の `PaywallSheet`）

Feature 行を以下に差し替え:

- 💬 話題リスト全カテゴリを解放（40話題を追加）
- 🌱 全話題の「反応別パターン」（盛り上がった時・薄い時・NG例）
- 👥 お相手の登録が無制限に（無料版は3人まで）
- 🚫 広告が永続的に非表示
- 🔁 買い切りのみ。追加課金・サブスクなし
- 🔒 データはこれまで通り端末内にのみ保存

### ハードコード数の撤廃（`src/screens/Topics.tsx` 151行付近）

「残り22種の話題…」が固定文字列になっている。以下のように動的に計算した数へ置き換える:

```ts
const lockedCount = TOPIC_CATEGORIES.filter(c => !c.free).reduce((n, c) => n + c.topics.length, 0);
```

文言は「残り◯種の話題＋反応別パターン＋お相手無制限＋広告なしを解放」。
