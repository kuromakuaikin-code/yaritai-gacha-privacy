# 施策×収益ログ（ShisakuLog）v0.1 プロトタイプ

「この施策をやったら収益・DLがどう動いたか」を記録し、施策と数値変化を同じ時間軸で見る
個人開発者向けアプリ。iOS 17+ / SwiftUI + SwiftData / Swift Charts。完全ローカル・手入力。

## 画面構成

- **記録（ホーム）**: アプリ切替セグメント → 今日の数値入力（収益・DLの2フィールド＋保存、
  DAU・メモは折りたたみ）→ 未入力なら「昨日の分を入力」→ 直近の施策リスト。
  起動 → 収益入力 → DL入力 → 保存 の3タップで完了する導線。
- **グラフ**: 収益/DLの折れ線（7日/30日/90日切替）に、施策の実施日を
  オレンジの縦線＋ラベルでオーバーレイ。縦線付近のタップ（またはグラフ下のリスト）で
  施策詳細を開き、前後7日平均の比較を表示。
- **施策ノート**: カテゴリフィルタ付きの時系列一覧。詳細は 仮説 → 前後比較 → 振り返り。
  実施から14日たって振り返り未記入の施策に「振り返り未記入」バッジ。

## 前後比較の仕様

- 前 = 実施日の前日までの7日間の平均 / 後 = 実施日の翌日からの7日間の平均
- 数値がない日（欠損日・nil）は平均から除外。片側が全欠損なら「データなし」
- UI文言は「前後比較」に統一し、「効果」とは表現しない（因果は断定しない）

## セットアップ

### 方法A: XcodeGen（推奨）

```bash
brew install xcodegen
cd shisaku-log
xcodegen
open ShisakuLog.xcodeproj
```

### 方法B: 手動

1. Xcode → File → New → Project → iOS App
   - Product Name: `ShisakuLog`（表示名は後で「施策×収益ログ」に）
   - Interface: SwiftUI / Storage: **None**（SwiftDataはコード側で設定済み）
   - Minimum Deployment: iOS 17.0
2. 自動生成された `ContentView.swift` と `〜App.swift` を削除
3. `shisaku-log/ShisakuLog/` の `.swift` 全ファイルをプロジェクトにドラッグ（Copy items if needed）
4. ビルド（⌘R）

## テストデータ

記録タブ右上の「…」メニュー →「テストデータを投入」で、
アプリ2本（やりたいガチャ / 盆踊りナビ愛知）× 30日分の数値 × 施策3件が入る。
欠損日も意図的に混ぜてあり、前後比較の「欠損日は除外」も確認できる。
初回起動の空状態からも投入できる。二重投入は同名アプリの存在チェックで防止。

## ロジックの検証（macOSのコマンドラインで実行可）

前後比較・表示フォーマットなどの純粋ロジックは `ShisakuLog/MetricsMath.swift` に分離してあり、
アプリをビルドしなくても検証できる:

```bash
cd shisaku-log
swiftc ShisakuLog/MetricsMath.swift tools/MetricsMathTests.swift -o /tmp/metrics-math-tests
/tmp/metrics-math-tests   # ALL PASSED で終了コード0
```
