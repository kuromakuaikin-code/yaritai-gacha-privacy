# 盆踊りナビ愛知 — 運営マニュアル

サイトの目的：**「今週末、愛知の近所で開催される小さな盆踊りを見つけること」**
観光サイト的な有名イベント一覧ではなく、町内会・学区・神社・公園レベルのローカル情報を扱う。

## 構成

```
bon-odori/
├── index.html      一覧ページ（今日／今週末／市町村／近い順）
├── submit.html     投稿案内ページ（GoogleフォームURLを設定して使う）
├── events.json     ★イベントデータ本体（これだけ編集すればよい）
├── build.py        個別ページ生成スクリプト
├── e/              生成された個別イベントページ（直接編集しない）
├── sitemap.xml     生成される（直接編集しない）
├── drafts/         掲載前の候補メモ（サイトには表示されない）
└── ADMIN.md        このファイル
```

## 日常の運用（イベントの追加・修正・非公開）

**`events.json` を編集してコミットするだけ**（書式は同ファイル冒頭の `_schema` 参照）。

- 編集はスマホ・PCの GitHub Web 画面からでもOK（リポジトリで events.json を開く → ✏️ → Commit changes）
- コミットすると GitHub Actions（`.github/workflows/build-bon-odori.yml`）が個別ページと sitemap を**自動で再生成**する
- ローカルで確認したい場合のみ `cd bon-odori && python3 build.py` を手動実行

- **追加**: 配列に1件足す。`id` は英数字とハイフン（例 `nagoya-midori-narumi-gakku`）。URLになるので後から変えない
- **修正**: 該当項目を直し、`lastVerified` を今日の日付に更新
- **非公開**: `status` を `"hidden"` にする（データは残る。build.py が個別ページを自動削除する）
- **中止・延期**: 消さずに `eventStatus` を `"cancelled"` / `"postponed"` にして `statusNote` に理由を書く（中止情報こそ利用者に価値がある）
- **承認待ち**: 投稿されたがまだ確認できていないものは `status: "pending"` で入れておく（非公開のまま管理できる）

## 毎年のデータ流用

前年のデータは消さない。翌年は同じエントリの以下だけ更新する：

1. `dates` を新しい日程に
2. `lastVerified` を確認日に
3. 必要なら `eventStatus` を `scheduled` に戻す

過去日程のイベントは一覧に自動で表示されなくなるだけなので、シーズン外もそのままでよい。
`id` を変えないことで個別ページのURLが年をまたいで維持され、Google検索の評価も引き継がれる。

## 投稿の受付（Googleフォーム）

1. Googleフォームを作成。質問は submit.html の5項目に合わせる：
   - チラシ画像（ファイルアップロード・任意）
   - イベント名（必須）
   - 日時（必須・記述式）
   - 場所（必須・記述式）
   - 情報元（必須・記述式：回覧板／掲示板／主催者本人／URLなど）
   - 任意：あなたの立場（主催者／住民）、画像の掲載許可（主催者本人 or 許可を得ている）
2. フォームURLを `submit.html` 内の `const FORM_URL = ""` に貼る
3. 回答はスプレッドシートに溜まる → 下の「掲載判断」を経て events.json に手で転記する

## 掲載判断のルール（重要）

- **必ず一次情報（主催者・自治体・町内会・神社・施設の公式発表）を探して確認**してから掲載する
- 公式情報が確認できた → `posterType: "admin"`（運営確認済み）または `"organizer"`（主催者本人の投稿）
- 公式情報が見つからない → `posterType: "resident"`（住民投稿・未確認）として区別して掲載するか、`status: "pending"` で保留
- 競合サイト・まとめサイトで見つけた場合も、**そこからは転載しない**。入口としてだけ使い、公式情報を探し直す
- `desc`（紹介文）は公式発表・投稿・自分の確認に基づき**自分の言葉で**書く。他サイトの紹介文のコピー・言い換えは禁止
- 画像は投稿者が権利を持つか掲載許可が確認できた場合のみ `imageAllowed: true` にして掲載

### 禁止事項

- 競合サイトからの自動取得・一括コピー・スクレイピング
- 他サイトの紹介文の言い換え転載、独自のまとめ方の流用
- チラシ画像・写真・ロゴの無断掲載
- Googleマップ等の画面キャプチャ掲載（リンクはOK）
- 情報元を隠した掲載

## 情報収集のコツ（一次情報の探し場所）

- 名古屋市は**区役所サイトに学区盆踊りの日程一覧**を載せる区がある（名東区で確認済み）。各区の「イベント」「区政協力委員会」ページを探す
- 尾張旭市のように**市が地区盆踊りを取りまとめて公開**している自治体もある
- 各市町村の**広報紙7月号・8月号**のイベント欄・回覧板コーナー
- `drafts/research-aichi-2026.json` に収集済みの候補メモあり（掲載時は一次情報の再確認と desc の書き直しが必須）

## 仮データについて

現在の events.json は表示確認用の仮データ10件（`"sample": true`）。
実データに切り替えるときは仮データのエントリを削除するか `status: "hidden"` にする。
`sample: true` が1件もなくなると、一覧上部の「仮データ掲載中」の注意書きは自動で消える。
