X/Y プロットパラメータ総当たりで比較グリッドを生成

「プロンプトS/R」は最初の値をプロンプト内から検索し、各値に置換して比較します(例: 笑顔, 泣き顔, 怒り顔)。

接続先複数登録してワンタップ切替(Mac / Windows / クラウド)

ComfyUI側はCORS許可付きで起動してください:
python main.py --enable-cors-header "*"
ローカル(このマシン上のComfyUI)は http://127.0.0.1:8188。SafariはlocalhostへのhttpをブロックすることがあるためChrome推奨。RunPodはポート8188のプロキシURL(https://…proxy.runpod.net)を貼り付けます。

ワークフローComfyUIの「API形式で保存」したJSON

ComfyUIの設定で「Dev mode」を有効にすると「Save (API Format)」が出ます。エクスポートしたJSONをそのまま貼ればOK。

ワイルドカードプロンプト内で __名前__ と書くと1行ランダム選択

使える記法:
{a|b|c} ランダムに1つ選択
__hair__ ワイルドカード「hair」から1行選択
${x=__hair__} 選んだ結果を変数xに保存(出力にも展開)
${x} 保存した値を再利用 — 同じキャラの髪色を複数箇所で一致させる等、クラスっぽい使い方ができます

データ

設定・ワークフロー・ワイルドカードはこの端末のブラウザ内(localStorage)、画像はIndexedDBに保存されます。サーバーには何も送信されません(ComfyUIエンドポイントを除く)。
